## 1. 项目概述

一款"荆棘收割者"的割草游戏，玩家控制角色自动射击，在敌人潮中生存尽可能长的时间。核心特色：

- **6 种独立武器**：旋转飞刀、闪电链、地雷、自动手枪、环绕飞剑、随机落雷 — 各自拥有独立冷却和 AI
- **13 种属性技能**：移速、子弹速度、双发、穿透、速射、巨弹、伤害强化、爆炸、护盾、回血、死亡新星、吸血、药水强化
- **3 种元素技能**：火、冰、雷（联动引擎触发组合效果）
- **8 种技能进化**（超武）：满级技能可进化成更强形态
- **8 种技能联动**：特定技能组合触发额外效果（烈焰穿透、冰爆、电弧等）
- **8 种敌人类型**：追敌、冲锋、远程、自爆、盾牌、自爆虫、精英射手、Boss
- **3 种可破坏物**：木桶、箱子、矿石（可掉落金币/回血/炸弹/磁铁）
- **局外成长**：9 种天赋树 + 金币经济系统
- **武器升级发光特效**：Lv3 金色、Lv4 橙色、Lv5 红色发光 + 缩放

### 控制方式

| 平台 | 移动 | 射击/技能 |
|------|------|-----------|
| 键盘 | WASD / 方向键 | 空格 / J 键 |
| 鼠标/触摸 | 虚拟摇杆（左下角） | 右下角技能按钮 |
| 手柄 | 左摇杆 | 右扳机 |

### 游戏流程

```
首页（HomePage）→ 点击"开始游戏" → GamePage（游戏页面）
  → 加载资源 → startGame() → 游戏循环开始（playing）
  → 升级时暂停 → 选择技能（selectSkill）→ 继续
  → 死亡 → gameOver() → 金币结算 → 返回首页 or 重新开始
```

```

├─────────────────────────────────────────────────┤
│             游戏对象层 (纯 JS Class)              │
│  Player  Enemy  BulletPool  ExperienceOrb       │
│  WeaponBase (6个子类)  SkillSynergy             │

└─────────────────────────────────────────────────┘
```

### 核心设计原则

1. **纯 JavaScript 游戏对象**：Player/Enemy/WeaponBase 等都是普通 JS 类，不依赖 Vue 响应式
2. **Canvas 原生渲染**：所有游戏画面用 Canvas 2D 绘制，保证 60FPS
3. **双 Canvas 分层**：背景层（bgCanvas）+ 实体层（entityCanvas），背景只在相机移动超过 5px 阈值时重绘
4. **独立武器系统**：每个武器拥有独立冷却计时器，在武器自己的 `update()` 中处理逻辑，在 `render()`/`draw()` 中绘制
5. **集中配置**：所有可调数值集中在 [`GameConfig.js`](src/game/GameConfig.js)，修改平衡性只需改这一个文件
6. **对象池**：子弹使用 `Float64Array` 扁平数组对象池，避免 GC 抖动
7. **程序化音效**：所有音效通过 Web Audio API 合成（`OscillatorNode` + `Noise`），无外部音频文件

## 4. 游戏核心循环

定义在 [`useGameEngine.js`](src/composables/useGameEngine.js:1208) 的 `setupGameEngine()` 中，使用 [`GameLoop`](src/game/GameLoop.js) 以固定时间步长运行。

### 4.1 GameLoop 架构

```javascript
// GameLoop 是 raf 驱动的游戏循环
class GameLoop {
  constructor({ update, render, fps = 60 })
  start()    // requestAnimationFrame 开始循环
  stop()     // cancelAnimationFrame 停止
  pause()    // 停止 raf，保存状态
  resume()   // 重置 _lastTime = performance.now()，重启 raf
}
```

### 4.2 更新阶段（update，每帧执行）

```js
1. 手柄/摇杆输入 → player.setGamepadAxes / player.setJoystick
2. 游戏时间累加
3. 屏幕抖动更新（triggerShake）
4. Player.update(dt) — 移动、惯性、碰撞、残影、HP 恢复
5. Camera.update(dt, player.x, player.y) — Lerp 平滑跟随
6. 敌人生成（按动态间隔，由 WaveManager 管理）
7. Boss 生成检测（300 秒 / 600 秒）
8. 可破坏物生成（按间隔随机）
9. 玩家子弹发射（bulletFireTimer）
10. 潮汐事件更新（WaveManager.updateTides）
11. 所有敌人 Enemy.update(dt)
12. 子弹池更新 BulletPool.updateAll(dt)
13. 技能联动更新 SkillSynergy.update(dt)
14. 经验球更新（磁吸拾取）
15. HitFlash / DamageNumber / DeathParticle 更新
16. 远程敌人射击
17. 碰撞检测（子弹 vs 敌人、敌人 vs 玩家、敌人子弹 vs 玩家、拾取球、子弹 vs 可破坏物）
18. 清理死亡对象（cleanupObjects — 硬上限 200 敌人/200 球体/60 数字/100 粒子）
19. ★ 武器系统更新（每个武器独立 update）
20. UI 同步（每 0.5 秒）
21. 自动存档（每 2 秒）
```

### 4.3 渲染阶段（render，每帧执行）

```js
1. 背景 Canvas：仅在相机移动超过 5px 阈值时重绘（网格 + 背景）
2. 实体 Canvas：
   a. 清空画布
   b. 应用相机偏移 (ctx.translate(-camera.x, -camera.y))
   c. Y 排序绘制（玩家 + 敌人 — 按 y 坐标排序实现伪 3D 效果）
   d. HitFlash / 经验球 / 子弹 / 敌人子弹 / 伤害数字 / 粒子
   e. 技能联动特效（燃烧地面、闪电弧）
   f. ★ 武器系统绘制（w.draw + w.render）
   g. 恢复相机偏移
   h. 绘制暗角、潮汐消息、调试信息（FPS/敌人数/子弹数）
```

### 4.4 启动流程（startGame）

[`startGame()`](src/composables/useGameEngine.js:806) 的完整流程：

```js
1. 销毁旧 Player（如果存在）
2. 重置 SkillSynergy.reset() — 清除燃烧地面、闪电弧等全局静态数组
3. 创建新 Player 实例
4. ★ 调用 PersistenceManager.applyTalents(player) — 应用天赋到玩家属性
5. 重置所有数组：enemies[], orbs[], weapons[], destructibles[], ...
6. 重置模块变量：gameState='playing', gameTime=0, score=0, ...
7. 创建 BulletPool（300 容量）
8. 创建 Camera
9. 创建 Background
10. 启动 WaveManager
11. 创建 GameLoop 并 start()
```

## 5. 武器系统详解

### 5.1 架构

所有武器继承自 [`WeaponBase`](src/game/WeaponBase.js)，通过 `update()` / `draw()` / `render()` 三阶段模式与主循环协作：

| 方法 | 调用时机 | 用途 |
|------|----------|------|
| `update(dt, player, enemies, refs)` | 主循环 update（每个武器逐一调用） | 冷却计时、AI 逻辑、碰撞检测、坐标计算 |
| `draw(ctx, camera)` | 主循环 render（相机变换后） | 绘制特效（闪电弧线、地雷圈等无实体东西） |
| `render(ctx, player)` | 主循环 render（相机变换后） | 绘制实体（飞刀刀片、飞剑等需世界坐标的东西） |

**性能优化**：`BoomerangWeapon` 和 `OrbitSwordWeapon` 在 `update()` 中计算并缓存 `wx/wy` 世界坐标，`render()` 直接读取缓存值，避免重复三角函数计算。

**武器升级机制**：调用 [`upgrade()`](src/game/WeaponBase.js:94) 递增 `_level`，触发 `_onLevelUp()` 让子类应用升级效果（如增加数量、减少冷却、提升伤害等）。

### 5.2 发光特效

武器等级 ≥ 3 时自动获得发光特效，由 [`WeaponBase.getGlowConfig()`](src/game/WeaponBase.js:49) 返回，配置在 [`WEAPON_GLOW`](src/game/GameConfig.js:188)：

| 等级 | 发光模糊 | 颜色 | 缩放 |
|------|----------|------|------|
| Lv1-2 | 0px | 白色 `#ffffff` | 1.00× |
| Lv3 | 6px | 金色 `#f1c40f` | 1.10× |
| Lv4 | 14px | 橙色 `#f39c12` | 1.20× |
| Lv5 MAX | 24px | 红色 `#ff6b6b` | 1.30× |

### 6.3 技能抽取规则

[`rollSkillOptions()`](src/game/SkillDefs.js:201) 每次升级时抽取 5 个选项（从 v1.0.9 改为 5 个）：

1. **进化选项**（最多 1 个）：已满级且未进化的技能 → 超武
2. **武器选项**（补齐到配额）：未拥有的武器（解锁）/ 已拥有的武器（升级）。武器配额随机在 1~4 之间
3. **属性技能**（补齐）：未满级的属性技能随机抽取
4. **保底**：如果仍不足 5 个，用已满级技能填充

### 6.4 技能选择流程

[`selectSkill()`](src/composables/useGameEngine.js:1020) 处理技能选择：

- **进化技能**：属性超武 → 直接 `apply(player)` 修改玩家属性；武器超武 → 升级武器实例 + 额外属性
- **武器技能**：未拥有 → `new 武器实例()` 推送到 `player.weapons[]` 和模块级 `weapons[]`；已拥有 → `existing.upgrade()`
- **属性技能**：直接调用 `skill.apply(player)`

选择技能后设置 `showLevelUpMenu.value = false` 关闭升级菜单。

## 8. 敌人生成系统

### 8.1 动态难度公式

定义在 [`useGameEngine.js`](src/composables/useGameEngine.js:167)：

```javascript
// 生成间隔: 从 2.0s 递减到 0.4s
spawnInterval = max(0.4, 2.0 - gameSeconds * 0.02)

// 敌人 HP: 基础 40，每分钟 +20%
enemyHp = floor(40 * (1 + minutes * 0.2))

// 敌人速度: 基础 100，每分钟 +0.08
enemySpeed = 100 + floor(minutes * 0.08) + random(0, 50)

// 敌人伤害: 基础 8，每分钟 +5（前 60 秒最低伤害）
enemyDamage = 8 + floor(max(0, minutes - 0.5) * 5)
```

### 8.2 敌人类型

定义在 [`Enemy.js`](src/game/Enemy.js)，8 种类型：

| 类型 | 行为 | 特殊属性 |
|------|------|----------|
| `chaser` | 直冲玩家 | 默认类型，移速中等 |
| `charger` | 接近后突然冲锋 | 冲锋速度 ×2，持续 2 秒 |
| `ranger` | 保持距离射击 | 射击间隔 1.5~2.5s，子弹速度 200 |
| `suicider` | 高速追敌后爆炸 | 爆炸半径 100px，伤害 30 |
| `shield` | 正面免疫子弹 | 盾牌 90° 免疫角度，血量 ×2.5 |
| `suicide_bug` | 靠近后变红 2 秒爆炸 | 高速，爆炸半径 90px |
| `elite_ranger` | 站桩发射慢速高伤飞行物 | 子弹速度 150，伤害 12 |
| `boss` | 多阶段巨型敌人 | HP ×20，3 阶段 AI，掉落超级宝箱 |

**Boss 阶段**：
- 第 5 分钟（300s）：第一个 Boss 出现

- Boss 有 3 个阶段：阶段 3 时发射扇形三发子弹

### 8.3 敌人死亡掉落

[`spawnEnemyDrops()`](src/composables/useGameEngine.js:988) 在敌人死亡时生成掉落物：

1. **基础经验球**（必掉）：`baseExp = 20`，可被 `premiumExpPer30s` 加成
2. **回血球**（概率）：基础概率 5%，每分钟 +2%，最高 20%。恢复量 = `15 + minutes * 5`
3. **高级经验球**（概率）：基础概率 2%，每分钟 +1.5%，最高 25%。经验值 = `50 + minutes/0.5 * 10`

## 11. 金币获取算法详解

金币系统是游戏中最重要的局外成长资源，用于升级天赋。金币获取有两种来源：**游戏结算**（主要）和 **可破坏物掉落**（辅助）。

### 11.1 来源一：游戏结算（主要金币来源）

当游戏结束时（玩家死亡或手动退出），[`saveRunProgress()`](src/composables/useGameEngine.js:915) 进行金币结算：

```javascript
// 结算公式
baseGold = Math.floor(score / 100) + player.kills
earnedGold = Math.floor(baseGold * player.goldMultiplier)

// 累加到总金币
PersistenceManager.addGold(earnedGold)
```

**变量说明**：

| 变量 | 来源 | 默认值 | 说明 |
|------|------|--------|------|
| `score` | `Player.addExp()` 累积 | 0 | 经验值 × 5（每次获得经验时 `this.score += amount * 5`） |
| `player.kills` | 击杀计数器 | 0 | 本局击杀的所有敌人数量 |
| `player.goldMultiplier` | 理财专家天赋 | 1.0 | 每级 +0.2（满级 5 级 → ×2） |

**结算示例**：

| 场景 | score | kills | goldMultiplier | baseGold | earnedGold |
|------|-------|-------|----------------|----------|------------|
| 新手局 | 500 | 20 | 1.0（无天赋） | `floor(500/100)+20 = 25` | `floor(25×1.0) = 25` |
| 中期局 | 3000 | 80 | 1.4（lv2理财） | `floor(3000/100)+80 = 110` | `floor(110×1.4) = 154` |
| 后期局 | 10000 | 200 | 2.0（lv5理财） | `floor(10000/100)+200 = 300` | `floor(300×2.0) = 600` |

**防重复结算**：使用 `_goldAlreadySaved` 标志位，确保 `saveRunProgress()` 在一局游戏中只执行一次金币结算。

### 11.2 来源二：可破坏物掉落（辅助金币来源）

可破坏物（木桶、箱子、矿石）被子弹摧毁时，通过 [`DestructibleObject.spawnDrops()`](src/game/DestructibleObject.js:113) 掉落金币：

```javascript
// 在 checkBulletDestructibleCollisions 中调用
dest.spawnDrops({
  onGoldDrop: (amount) => {
    PersistenceManager.addGold(amount * DESTRUCTIBLE_CONFIG.goldPerDrop)
    // goldPerDrop = 1 (定义在 GameConfig.js)
  },
  // ... 其他回调
})
```

**可破坏物掉落表**：

| 类型 | 可掉落的物品 | 权重 |
|------|-------------|------|
| 木桶（barrel） | 金币(1-2)、回血(15-25)、炸弹、无 | 各自权重随机 |
| 箱子（crate） | 金币(2-5)、回血(20-35)、磁铁、无 | 金币掉率较高 |
| 矿石（ore） | 金币(3-8)、回血(30-50)、炸弹、磁铁 | 高价值，较稀有 |

**金币计算公式**：

```
实际获得金币 = 掉落表中的随机数量 × DESTRUCTIBLE_CONFIG.goldPerDrop
```

由于 `goldPerDrop = 1`，所以掉落表中的金币数量直接就是获得的金币数。

**注意**：可破坏物掉落的金币不受 `goldMultiplier` 加成，是纯粹的额外金币。

### 11.3 金币消耗

金币唯一的消耗途径是天赋升级：

```javascript
// PersistenceManager.upgradeTalent(talentId)
cost = talent.cost[currentLevel]  // 从费用数组中取当前等级对应费用
spendGold(cost)                   // localStorage 中减去金币
```

天赋总费用表：

| 天赋 | 满级总费用 |
|------|-----------|
| 生命强化（lv5） | 100+200+400+800+1600 = **3,100** |
| 敏捷（lv3） | 150+300+600 = **1,050** |
| 攻击强化（lv5） | 200+400+800+1600+3200 = **6,200** |
| 护盾强化（lv3） | 300+600+1200 = **2,100** |
| 缓慢恢复（lv5） | 200+400+800+1600+3200 = **6,200** |
| 弹道加速（lv3） | 150+300+600 = **1,050** |
| 理财专家（lv5） | 400+800+1600+3200+6400 = **12,400** |
| 防御强化（lv10） | 300+500+800+1200+1700+2300+3000+3800+4700+5800 = **25,700** |

### 11.4 经验 vs 分数的关系

分数（score）间接影响金币获取，其算法为：

```javascript
// Player.addExp(amount) 中
this.exp += amount
this.score += amount * 5  // 经验值 × 5 = 分数
```

- 每个基础经验球 = 20 exp → 100 分
- 每个高级经验球 = 50 exp（随时间增长）→ 250+ 分
- 每次升级不会直接给分数，但击杀敌人获得的经验球会累积

### 13.1 类型定义

```javascript
DESTRUCTIBLE_TYPES = {

  crate:  { hp: 4, radius: 24, color: '#CD853F', dropTable: [...] },
  ore:    { hp: 8, radius: 28, color: '#808080', dropTable: [...] },
```

### 13.2 掉落表（dropTable）

每个可破坏物类型有 `dropTable`，包含加权掉落条目：

```javascript
// 以箱子为例
[
  { type: 'gold',   weight: 40, min: 2, max: 5   },
  { type: 'heal',   weight: 25, min: 20, max: 35  },
  { type: 'magnet', weight: 15, duration: 5        },
  { type: 'nothing', weight: 20                    },
]
```

**掉落机制**：

```javascript
function rollDrop(dropTable) {
  const totalWeight = dropTable.reduce((sum, e) => sum + e.weight, 0)
  let roll = Math.random() * totalWeight
  for (const entry of dropTable) {
    roll -= entry.weight
    if (roll <= 0) return entry
  }
  return dropTable[dropTable.length - 1]
}
```

### 13.3 金币掉落处理

金币掉落时，通过回调将金币直接加到 `PersistenceManager`（局外总金币），不受游戏状态影响：

```javascript
spawnDrops(refs) {
  const chosen = rollDrop(this.typeDef.dropTable)
  if (chosen.type === 'gold') {
    const amount = chosen.min + Math.floor(Math.random() * (chosen.max - chosen.min + 1))
    refs.onGoldDrop(amount)  // → PersistenceManager.addGold(amount * goldPerDrop)
  }
  // ... heal / bomb / magnet 处理
}
```

- 最大数量：同时最多 15 个
- 超出屏幕一定距离后自动标记为死亡

## 14. 子弹对象池系统

定义在 [`BulletPool.js`](src/game/BulletPool.js)。

### 14.1 数据结构

```javascript
class BulletPool {
  constructor(poolSize = 300) {
    // 扁平 Float64Array: poolSize × 7 个槽位
    // 每颗子弹占用 7 个 double：x, y, dirX, dirY, speed, radius, penetration
    this._data = new Float64Array(poolSize * 7)

    // 存活标志：1=存活，0=死亡
    this._alive = new Uint8Array(poolSize)

    // 存活索引缓存（存活子弹的 index 列表）
    this._aliveIndices = new Int32Array(poolSize)
    this._aliveCount = 0
  }
}
```

**为什么用 Float64Array 而不是对象数组？**

- 避免 GC 分配：游戏运行时不会创建/销毁子弹对象
- 内存紧凑：连续内存布局，缓存友好
- 批量更新：`updateAll()` 遍历时仅访问 `_data` 和 `_alive` 数组

### 14.2 核心方法

| 方法 | 用途 |
|------|------|
| `allocate(x, y, dirX, dirY, speed, penetration, radius)` | 分配一颗新子弹（使用 `_oldestAliveIndex` 回收最旧子弹） |
| `deallocate(index)` | 将子弹标记为死亡 |
| `setPenetration(index, pen)` | 直接修改子弹的穿透值（写入 `_data[index*7+6]`） |
| `get(index)` | 返回子弹快照对象 `{ index, x, y, dirX, dirY, speed, penetration, radius }` |
| `getRaw(index)` | 返回包含 `_data` 引用的可写对象（用于武器系统直接写入） |
| `updateAll(deltaTime)` | 更新所有存活子弹位置，回收超出边界的子弹 |
| `forEach(fn)` | 遍历所有存活子弹，回调接收快照对象 |
| `reset()` | 清空所有子弹 |

### 14.3 关键注意事项

**`forEach()` 的快照问题**：`forEach()` 创建的是临时快照对象，修改快照的 `penetration` 属性不会影响池中数据。必须使用 `setPenetration()` 或 `deallocate()` 来修改。

**旧子弹回收**：当池满时，`allocate()` 会调用 `_oldestAliveIndex()` 找到最早存活的子弹进行回收，而不是直接拒绝新子弹。

## 16. 全局暂停/恢复管理

系统实现了全局暂停/恢复，确保暂停时游戏完全停止。

### 16.1 GameLoop 暂停

[`GameLoop.pause()`](src/game/GameLoop.js:45)：

```javascript
pause() {
  cancelAnimationFrame(this._rafId)  // 停止 rAF
  this._paused = true
}
```

[`GameLoop.resume()`](src/game/GameLoop.js:55)：

```javascript
resume() {
  this._lastTime = performance.now()  // 重置时间基准，防止 dt 跳跃
```

**关键**：`resume()` 重置 `_lastTime` 为当前时间，防止暂停期间的时间差导致 dt 异常，避免玩家恢复后看到"时间跳跃"。

## 19. 已知 Bug 修复记录

### 19.1 [关键] 子弹穿透无限 Bug — 已修复

**问题描述**：`checkBulletEnemyCollisions()` 中使用 `toDeallocate[]` 批处理数组收集子弹穿透变化，然后在 `forEach()` 循环结束后统一处理。但 `BulletPool.forEach()` 传递的是快照对象，快照的 `penetration` 属性在循环中不会更新。这导致穿透值 > 1 的子弹在一个渲染帧内可以无限穿透所有敌人。

**修复方案**（[`useGameEngine.js`](src/composables/useGameEngine.js:517)）：

```javascript
// 修复前：使用批处理数组，穿透值不实时更新
const toDeallocate = []
bulletPool.forEach(bullet => {
  // ...
  if (bullet.penetration > 1) {
    toDeallocate.push({ index: bullet.index, dealloc: false, penLeft: bullet.penetration - 1 })
  } else {
    toDeallocate.push({ index: bullet.index, dealloc: true })
  }
  if (bullet.penetration <= 1) break
})
// 批处理
for (const item of toDeallocate) { ... }

// 修复后：立即调用池方法更新
bulletPool.forEach(bullet => {
  // ...
  const penLeft = bullet.penetration - 1
  if (penLeft > 0) {
    bulletPool.setPenetration(bullet.index, penLeft)  // 直接写入 Float64Array
  } else {
```

### 19.3 [中等] restartGame 不关闭升级菜单 — 已修复

### 19.4 [关键] SkillSynergy 静态数据内存泄漏 — 已修复

**问题描述**：`SkillSynergy.firePools[]` 和 `SkillSynergy.chainLightnings[]` 是静态数组，在 `startGame()` 中没有被清空。多局游戏后，这些数组累积了之前局次的燃烧地面和闪电弧数据，导致性能下降。

```javascript
  SkillSynergy.chainLightnings = []
}
```

### 19.5 [信息] AudioContext 自动播放限制 — 已修复

**问题描述**：浏览器要求在用户交互后才能创建/恢复 AudioContext。首次加载页面时自动播放音效会因"not allowed to start"错误失败。

**修复方案**（[`AudioManager.init()`](src/game/AudioManager.js:40)）：

```javascript
init() {
  this._ctx = new AudioContext()
  // 监听首次用户交互事件，恢复 AudioContext
  const resumeOnInteraction = () => {
    if (this._ctx.state === 'suspended') {
      this._ctx.resume()
    }
  }
  window.addEventListener('click', resumeOnInteraction, { once: true })
  window.addEventListener('touchstart', resumeOnInteraction, { once: true })
  window.addEventListener('keydown', resumeOnInteraction, { once: true })
}
```

### 19.6 [信息] 虚拟摇杆移动端修复 — 已修复

**问题描述**：移动端虚拟摇杆存在定位问题和触摸事件冲突，以及静态模式在页面滚动后位置不准确。

**修复方案**：重写 `VirtualJoystick` 类，改用 `touch` 事件（`touchstart`/`touchmove`/`touchend`），调整相对坐标计算逻辑，增加动态/静态两种模式切换。

### 21.6 完整的 Git 工作流程

```
克隆仓库 → 安装依赖 → 开发 → 提交 → 推送到远程
```

#### 首次设置

```bash
# 1. 克隆
git clone https://github.com/username/thorn-reaper.git
cd thorn-reaper

# 2. 安装
npm install

npm run dev
```

#### 日常开发流程

```bash
# 1. 拉取远程最新代码
git pull origin main

# 2. 创建功能分支
git checkout -b feature/my-new-feature

# 3. 在本地修改代码（Vite 热更新自动生效）

# 4. 查看修改的文件
git status

# 5. 添加修改到暂存区
git add .

# 6. 提交到本地仓库
git commit -m "feat: 添加 xxx 功能"

# 7. 推送到远程仓库
git push origin feature/my-new-feature

# 8. 在 GitHub 上创建 Pull Request 合并到 main 分支
```

#### 从远程拉取更新

```bash
# 拉取远程 main 分支的最新代码到本地
git pull origin main

git stash            # 暂存本地修改
git pull origin main # 拉取远程更新
git stash pop        # 恢复本地修改
```

### 21.7 常见问题排查

**Q：`npm install` 报错 / 安装失败？**

A：尝试以下步骤：
```bash
# 清除 npm 缓存
npm cache clean --force

# 删除 node_modules 重新安装
rm -rf node_modules package-lock.json
npm install
```

**Q：`git clone` 很慢或失败？**

A：可以使用国内镜像或更改协议：
```bash
# 使用 SSH 协议（需先配置 SSH Key）
git clone git@github.com:username/thorn-reaper.git

# 或者使用代理（如果有）
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890
```

**Q：启动开发服务器后浏览器白屏 / 无法访问？**

A：检查以下常见原因：
1. 确认终端显示的是 `http://localhost:5173` 而非 `0.0.0.0`
2. 检查防火墙是否阻止了 Node.js
3. 尝试用 `npx vite --host 0.0.0.0` 允许局域网访问

**Q：`git push` 提示权限不足？**

A：确保你已添加到仓库的 Collaborator 列表，或使用 SSH Key 认证：
```bash
# 生成 SSH Key（如果还没有）
ssh-keygen -t ed25519 -C "your_email@example.com"

# 查看公钥，复制后添加到 GitHub Settings → SSH and GPG keys
cat ~/.ssh/id_ed25519.pub
```
